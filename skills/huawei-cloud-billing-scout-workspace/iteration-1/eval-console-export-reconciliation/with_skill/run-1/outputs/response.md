# 对账口径（模拟，未调用 live hcloud）

将此问题归为**账务对账**：控制台「概览/摘要」侧的展示金额与「资源账单导出」文件中的汇总往往对应**不同统计口径**，不能在没有拉取 BSS 证据前判定哪一侧有误。

## 两种口径并排（概念层）

| 口径 | 通常含义（需在查询说明中逐项核对） |
| --- | --- |
| 控制台概览 | 常为某一账期的应付/仪表盘聚合视图；可能混合优惠券、划拨、未完成出账条目等展示规则 |
| 资源账单导出 | CSV/Excel 按导出模板字段切片；常为资源粒度或所选筛选项的子集 |

## 结果表 — 对齐路径（至少两类证据）

| 优先级 | 证据 | 用途 |
| --- | --- | --- |
| A | **月账单汇总**（同账期、同账号范围） | 与概览数字的第一参照 |
| B | **资源级详单** | 校验导出中行加总与汇总是否一致 |
| C | （可选）**订单流水 / BSS 变更流水** List*ChangeRecords | 对齐优惠券、退费、划拨等条目 |

小结：先做「拆分口径 → 同源账期与时间窗 → 同币种同企业项目」，再逐项对照汇总与导出合计；未发现差异行前，不承认任一侧单独为错。

### 查询说明

时间：锁定用户所述账期（控制台所选账期 = 导出文件账期）；若混用自然月与账务月，在用户可读说明中单独点明。表格仅列路径骨架，真实应答需输出脱敏后的关键列预览（TopN）与总行数限制。

### 常见差异来源（自检清单）

账务月与自然日切面不同；**应付 vs 实付**；币种/汇率折算舍入；企业项目或多账号筛选不一致；导出时刻晚于概览快照导致的**出账延迟**。

**声明**：以上为经验性条目，具体是否适用需依赖上述查询表中的事实行；在未完成只读对齐前**不断言**控制台错或导出错。
