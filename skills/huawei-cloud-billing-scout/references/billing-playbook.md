# 账务协作手册

本手册帮助 Agent 把用户问题转成最小证据链：先判断协作目标，再选择事实实体、
查询顺序和输出形态。不要把用户话术当固定问法模板匹配；同一句话可能同时包含事实、
归因、对账或咨询。

## 1. 分流

查事实：用户要一个当前状态、金额、余额、用量或 TopN。
典型信号：余额、欠费、本月/上月花费、资源/天费用、企业项目、子账号费用。
首查 `AccountBalance`、`MonthlyBillSummary`、`ResourceBillDetail` 或
`CostAnalysis`；余额或月汇总优先，多维费用用 `ListCosts`。

归因：用户要解释“为什么”或“钱去了哪里”。
典型信号：一直扣钱、删了还扣、哪项最高、资源包/代金券未抵扣、费用变化。
首查 `ResourceBillRecord`、`CostAnalysis`、`FreeResourcePackage` 或
`CouponChangeRecord`；抵扣问题先查包/券流水。

对账：用户拿两套数字或两种视图要求解释差异。
典型信号：概览 vs 导出、汇总 vs 明细、订单 vs 消费、API vs 控制台金额不一致。
首查月汇总、详单、订单/流水证据；拆两边口径，分别查询，再对齐差异。

咨询：用户要规则、路径或下一步，而不是单纯查数。
典型信号：导出、PDF/盖章、权限、账期、欠费、分摊规则、退款/代付路径。
能查则先给只读证据；操作类引导控制台、工单或账号经理。

这些信号只是校准例子，不是穷举匹配表。路由仍以事实粒度、维度、指标和
证据链为准。

## 2. 口径

先写清：时间(账期/自然日/近 N 天/跨月)、范围(当前账号/企业主/子账号/支付账号)、维度(云服务/资源/区域/企业项目/计费模式/账单类型)、粒度(月/日/记录/详单)、金额(应付/实付/现金/授信/券/储值/欠费)。

有安全默认先写再查；会扩大账号范围或改变问题含义时先问。

## 3. 查询路径

| 目标 | 顺序 |
| --- | --- |
| 余额/欠费 | 余额 -> 月汇总 -> 资源记录 |
| 月花费 | 月汇总 -> 成本分析 -> 详单 |
| 持续扣费 | 资源记录 -> 详单 -> 产品只读核验 |
| 资源包未抵扣 | 包列表 -> 余量 -> 抵扣明细 -> 按需详单 |
| 代金券未抵扣 | 券流水 -> 月汇总构成 -> 详单 |
| 对账不一致 | 两边口径 -> 分别查 -> 对齐 -> 列差异 |
| 多账号/企业项目 | 授权判断 -> `method`/范围 -> 汇总 -> 明细 |

持续扣费默认查 7/14/30 天记录，按 `amount`、`consume_amount`
或应付金额降序聚合，再取 Top 详单。结论只落在三类：
可见资源仍计费、账期历史费用、BSS 有账但产品侧未核到资源。

对账必须列两边口径。常见差异：汇总 vs 详单(8 位小数/分扣)、
订单 vs 消费(包周期/按需)、控制台 vs 导出(延迟/筛选/企业项目/账期)、
账期 vs 自然日、现金 vs 应付(券/储值/授信/调账)、
企业项目(default/未归集/不回刷)。

## 4. 追问与拒绝

默认不追问：本月/上月/最近、当前认证 profile、缺资源 ID 的 TopN 排查。
必须追问：企业范围不明且查询会扩大到 `all`、用户要查他人账单、时间/金额/资源口径会改变结论。
必须拒绝执行：删/关资源、支付、续费、退款、改余额、代付、导出未授权账单；改为只读证据清单和官方处理路径。

## 5. 表格列

月汇总：账期、云服务、资源类型、计费模式、账单类型、应付、现金、券、欠费。
资源扣费：日期/账期、云服务、资源名、ID 尾号、区域、企业项目、用量类型、金额。
资源详单：账期/日期、云服务、资源 ID 尾号、区域、企业项目、计费模式、账单类型、金额构成。
资源包：名、状态、生效/失效、资源项、余量、抵扣用量。
代金券：券 ID 尾号、时间、收支、交易类型、交易 ID 尾号。
多账号：账号尾号、云服务、企业项目、金额、占比。

ID 脱敏：账号、资源、订单、交易只显示前后 4-6 位或尾号。

## 6. 输出原则

账务回答的核心顺序是证据先于结论，而不是机械套固定模板。简单事实查询可以
用一张表和一句说明结束；归因、对账和大结果才展开范围、差异和边界。

- 结果表：优先出现，ID 脱敏；没有可查询证据时说明无法下结论。
- 小结：叙事性直接答问，写数据依据、假设、限制，不复述表格。
- 用户可读说明：标题随意图选择：查询说明/统计范围/对账口径/数据来源；正文用自然语言写清时间、账号范围、区域、筛选、排序、金额口径。
- 待核验边界：仅列影响结论但当前证据无法确认的事实，不猜责任方，不把常规限制写成待办。
- 技术细节默认隐藏：API 名、状态码、字段名、命令摘要不进对话；仅复现、排错、审计或本地报告追加。
- 大结果分层呈现：对话只给 TopN + 固定结构；全量结果写本地报告。
