# 计费协作手册

这里写判断原则，不写仪式化流程。结论必须贴住查询证据。

## 路由

- 余额、欠费、可用额度：
  首查 `AccountBalance`；必要时下钻 `MonthlyBillSummary`、账户流水。
- 本月/上月花了多少：
  首查 `MonthlyBillSummary`、`CostAnalysis`；必要时查资源详单、摊销成本。
- 一直扣钱、删了还扣：
  首查 `ResourceBillRecord`；必要时查详单行、产品侧只读核验。
- 资源包/券/储值卡没抵扣：
  首查 `FreeResourcePackage`、`CouponChangeRecord`、`StoredValueCard`；
  必要时下钻账单详单、使用量记录、账户流水。
- 控制台/导出/API 对不上：
  首查 `BillingStatement`、`MonthlyBillSummary`；必要时查订单证据、账户流水。
- 企业/子账号费用：
  首查 `EnterpriseAccountContext`、`EnterpriseBilling`；
  必要时查范围确认后的客户明细。
- 伙伴/代售客户费用：
  首查 `PartnerAccountContext`、`PartnerBilling`；
  必要时查授权证据后的客户账单明细。
- 报价、续订金额：
  首查 `PricingQuote`；用户问实际扣费时回到账单/订单事实。

## 关键口径

- 账期是账务归集月份，不一定等于自然使用日期。
- 应付是账单义务；实付是现金、授信、券、储值卡、欠费等支付构成。
- 成本分析是聚合；详单行是消费证据；账户流水是结算证据。
- 资源详单可能保留比分扣款更高的小数精度，不要强行逐行相等。
- 产品 API 不能抹掉历史账单。“当前未查到资源”只说明当前产品侧查不到。
- 企业项目 `default`、`0`、`null` 和“未归集”不能混用。
- 价格试算和折扣策略不是发票，也不是已出账金额。

## 默认范围

- 时间：“本月”取当前月；“上月”取最近完整月；“最近”默认近 7 天，除非接口要求账期。
- 账号：默认当前 KooCLI profile、`method=oneself`。只有用户明确要求时才用 `all` 或伙伴客户列表。
- 金额：用户未指定时，展示应付金额；可得时同时展示支付构成。
- 粒度：先汇总，再详单；先账务事实，再产品侧核验。

## 红线

拒绝写操作和有副作用的操作。改为提供只读证据和官方处理路径。

例子：支付订单、续费资源、退订/退款、回收券或余额、修改账户金额、创建子客户、发送验证码、修改续费配置、删除/停用资源。

## 大结果

不要粘贴全量结果。按能回答问题的维度聚合，展示 TopN 和少量样例。导出全量明细前先确认。评测和审阅产物只保存脱敏摘要。

## 对账

必须说清正在对齐的两边：

- 汇总 vs 明细
- 订单 vs 消费
- 账期 vs 自然日期
- 应付 vs 实付
- 原始成本 vs 摊销成本
- 当前资源状态 vs 历史账单
- 当前账号 vs 企业/伙伴范围

任一侧缺证据时，只能给假设，不能给定论。
