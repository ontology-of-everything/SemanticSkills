0. 30 秒定位

结论：两者不是替代关系。skills-ref 是规范层的官方最小契约；skillcheck 是在同一 spec 之上的扩展质量门禁。你们 pipeline 先跑 skills-ref validate 再跑 skillcheck，正是「先过 schema，再过 lint」。

1. 功能对比总表

能力 skills-ref skillcheck 说明
SKILL.md 存在 ✓ ✓ skills-ref 也接受 skill.md 小写
YAML frontmatter 解析 ✓ ✓ ref: 手写 split + js-yaml；check: PyYAML
name / description 必填 ✓ ✓ 共有
name ≤64、description ≤1024 ✓ ✓ 与 agentskills.io 一致
目录名 == name ✓ ✓ check 对 VS Code 更强调（可 --strict-vscode）
name 字符集 Unicode 字母 + 数字 + - 仅 [a-z0-9-] 分歧：ref 支持 i18n name；check 更严 ASCII kebab-case
frontmatter 字段白名单 严格 6 字段 spec + ecosystem + toml 扩展 check 允许 version、hooks 等 Agent 扩展字段
description 质量评分 — ✓ 0–100 动词 / 触发语 / 关键词 / 具体性 / 长度五维
body 行数 / token 预算 — ✓ 500 行、8000 tokens；metadata ~100、body ~5000
文件引用完整性 — ✓ 断链、越界路径、引用深度
跨 Agent 兼容矩阵 — ✓ Claude / VS Code / Codex / Cursor
生成 <available_skills> XML ✓ to-prompt — ref 独有：给 Agent system prompt 用
read-properties JSON ✓ — ref 独有
Capability graph / history — ✓ check 进阶：图分析、历史 ledger
LLM critique 接入 — ✓ 可选 emit/ingest critique prompt（非默认）
扫描 scripts 安全 — — 两者都不做；用 cisco-ai-skill-scanner

2. 实现架构对比

skills-ref 实现栈

  • 语言：TypeScript (Node ≥14)
  • 包：npm skills-ref@0.1.5
  • 依赖：js-yaml、commander
  • 入口：dist/cli.js
  • 核心模块： parser.js → validator.js → prompt.js
  • 体量：~60KB，31 文件
  • 定位：README 写明 demonstration / reference only

skillcheck 实现栈

  • 语言：Python ≥3.10
  • 包：pip skillcheck@1.3.0
  • 依赖：pyyaml；可选 tiktoken
  • 入口：skillcheck.cli:main
  • 核心模块： parser → rules/* → formatters → core/graph
  • 体量：744 tests，多 rule 模块
  • 定位：Production/Stable cross-agent quality gate

3. Frontmatter 校验：重叠与分歧

3.1 共有规则（实现几乎同构）

检查项skills-refskillcheck rule id
frontmatter 以 --- 包裹parseFrontmatter解析阶段
name 非空、lowercasevalidateNamefrontmatter.name.*
无首尾/连续 hyphen✓leading-trailing-hyphen · consecutive-hyphens
description 非空、≤1024validateDescriptionfrontmatter.description.*
compatibility ≤500✓（在 spec 字段集中）
目录名匹配 nameNFKC 后比较frontmatter.name.directory-mismatch

3.2 关键分歧（源码证据）

skills-ref：严格 6 字段白名单

ALLOWED_FIELDS = {
  "name", "description", "license",
  "allowed-tools", "metadata",
  "compatibility"
}
// 多一个 key → error

agentskills.io spec 对齐

skillcheck：分层字段模型 + toml 扩展

SPEC_FIELDS      # version, hooks, agent...
ECOSYSTEM_FIELDS # license, repository...
extension_fields # skillcheck.toml 声明
# unknown → info/warning，非一律 error

你们仓库 skillcheck.toml：
extension_fields = ["license", "compatibility", "metadata"]

skills-ref：name 允许 Unicode

// 支持 Cyrillic、CJK 等 i18n
isLetter(c) // Latin + Cyrillic + CJK...

skillcheck：ASCII kebab-case only

_NAME_VALID_CHARS_RE =
  re.compile(r"^[a-z0-9-]+$")

与 agentskills.io 文档字面一致；ref 更宽

skillcheck 额外 frontmatter 规则（ref 无）

  • description.no-xml-tags — 禁止 description 含 < >（防 XML 注入 to-prompt）
  • description.person-voice — 人称/语气建议
  • name.reserved-word — 含 anthropic/claude 警告
  • yaml.anchors — 禁止 YAML anchor 混淆
  • template.detected — 识别模板占位 skill

4. skillcheck 独有能力（实现摘要）

4.1 Description 质量分（核心差异化）

五维加权，总分 100 — 源码 rules/description.py：

维度满分测什么
action25是否以动作动词开头（200+ 动词表 + 三单归一化）
trigger25「Use when…」「Make sure to use this skill…」等触发语
keywords25内容词 / 停用词比例（反 filler）
specificity15 vague 词（tool/helper/stuff…）密度
length1040–300 字符 sweet spot
 
skillcheck "$SKILL_DIR" --min-desc-score 70
# → score < 70 时 WARNING（description.min-score）

4.2 Progressive disclosure 预算

源码 rules/disclosure.py + config.py：

  • metadata 层 ~100 tokens（name + description）
  • body 层 ~5000 tokens（SKILL.md 正文）
  • bloat 检测：代码块 >50 行、表格 >20 行

4.3 文件引用图

rules/references.py：解析 Markdown 链接 / run 语句 / import，验证目标文件存在、不逃逸 skill 目录、深度限制。

4.4 跨 Agent 兼容矩阵

config.COMPAT_MATRIX + rules/compat.py：每个 frontmatter 字段在 claude / vscode / codex / cursor 下是 supported / ignored / unknown。

--target-agent cursor --strict-cursor
# Claude-only 字段（hooks, mode, agent...）在 Cursor 下告警/报错

5. skills-ref 独有能力

5.1 validate — 最小 schema _gate

skills-ref validate path/to/skill
# exit 0: "Valid skill: ..."
# exit 1: 列出 validateMetadata errors

5.2 read-properties — 结构化读取

skills-ref read-properties path/to/skill
# → JSON { name, description, license, ... }

5.3 to-prompt — Agent 集成

生成 Anthropic 推荐的 <available_skills> XML，含 name、description、SKILL.md 绝对路径；HTML escape 防注入。

<available_skills>
<skill>
<name>my-skill</name>
<description>...</description>
<location>/path/to/SKILL.md</location>
</skill>
</available_skills>

源码：dist/prompt.js · agentskills.io 文档直接推荐此工具做 validate

6. 你们 CI 中的顺序

# template/qa/validate.sh
run_local_or_npx skills-ref validate "$SKILL_DIR"
run_local_or_npx markdownlint-cli2 "$SKILL_DIR/**/*.md"
skillcheck "$SKILL_DIR" --target-agent cursor --strict-cursor --min-desc-score 70
阶段工具失败含义
1skills-ref不符合 agentskills.io 最小 spec → 平台可能无法注册 skill
2markdownlintMarkdown 风格不一致
3skillcheck质量/兼容/description 分/引用/体积问题
4+scanner / evals安全 / 运行时行为

7. 选型建议

场景用哪个
官方 spec 合规、生成 available_skills XMLskills-ref
CI 质量门禁、多 Agent 发布、description 可发现性skillcheck
只需回答「frontmatter 合法吗」skills-ref 足够
Cursor strict + desc score ≥70必须 skillcheck
两者都跑推荐 — 重叠成本低，互补价值高
实现视角一句话： skills-ref 是一个单函数 schema validator（~200 行核心逻辑 + prompt 工具）； skillcheck 是一个可配置的 rule engine（30+ 规则函数、compat 矩阵、scoring rubric、可选 graph/critique）。 前者定义「能不能被 Agent 平台识别」；后者定义「识别之后好不好用、会不会在 VS Code/Cursor 上静默失败」。
